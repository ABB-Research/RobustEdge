# Data release guidance

For GitHub, do not commit full raw logs directly unless Git LFS is configured.

Recommended public release:

1. Code repository on GitHub.
2. Dataset archive via GitHub Releases, Zenodo, OSF, or institutional repository.
3. Manifest CSV generated by `scripts/make_manifest.py`.
4. Checksums for archives.
5. Tagged code release corresponding to paper submission or camera-ready version.
